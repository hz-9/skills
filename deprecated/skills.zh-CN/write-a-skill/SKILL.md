# 编写技能

## 流程

1. **收集需求** - 询问用户：
   - 该技能涉及什么任务/领域？
   - 应该处理哪些具体用例？
   - 需要可执行脚本还是只需要指令？
   - 是否包含任何参考资料？

2. **起草技能** - 创建：
   - 包含简洁指令的 SKILL.md
   - 如果内容超过 500 行，添加额外的参考文件
   - 如果需要确定性操作，添加工具脚本

3. **与用户复核** - 展示草稿并询问：
   - 这涵盖了你的用例吗？
   - 是否有遗漏或不清楚的地方？
   - 是否有章节需要更详细或更简略？

## 技能结构

```
skill-name/
├── SKILL.md           # 主要指令（必需）
├── REFERENCE.md       # 详细文档（如果需要）
├── EXAMPLES.md        # 使用示例（如果需要）
└── scripts/           # 工具脚本（如果需要）
    └── helper.js
```

## SKILL.md 模板

```md
---
name: skill-name
description: 能力的简要描述。当[特定触发条件]时使用。
---

# 技能名称

## 快速开始

[最小可用示例]

## 工作流程

[带有检查清单的分步流程，用于复杂任务]

## 高级功能

[链接到单独的文件：参见 [REFERENCE.md](REFERENCE.md)]
```

## 描述要求

描述是 **你的 agent 在决定加载哪个技能时看到的唯一信息**。它与其他所有已安装的技能一起出现在系统提示中。你的 agent 会读取这些描述，并根据用户的请求选择相关的技能。

**目标**：给你的 agent 提供足够的信息，让它知道：

1. 这个技能提供了什么能力
2. 何时/为什么触发它（特定关键词、上下文、文件类型）

**格式**：

- 最多 1024 个字符
- 用第三人称写
- 第一句话：它能做什么
- 第二句话："当[特定触发条件]时使用"

**好示例**：

```
从 PDF 文件中提取文本和表格、填写表单、合并文档。当处理 PDF 文件或用户提到 PDF、表单或文档提取时使用。
```

**坏示例**：

```
帮助处理文档。
```

坏示例让你的 agent 无法将这个技能与其他文档技能区分开。

## 何时添加脚本

在以下情况下添加工具脚本：

- 操作是确定性的（验证、格式化）
- 相同的代码会被重复生成
- 错误需要显式处理

与生成的代码相比，脚本可以节省 token 并提高可靠性。

## 何时拆分文件

在以下情况下拆分为单独的文件：

- SKILL.md 超过 100 行
- 内容包含不同的领域（例如财务模式与销售模式）
- 高级功能很少用到

## 复核检查清单

起草完成后，验证以下内容：

- [ ] 描述包含触发条件（"当...时使用"）
- [ ] SKILL.md 不超过 100 行
- [ ] 没有时效性信息
- [ ] 术语保持一致
- [ ] 包含具体示例
- [ ] 引用层次不超过一层
