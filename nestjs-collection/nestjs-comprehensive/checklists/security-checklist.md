# 安全检查清单

NestJS 应用程序的综合安全检查清单。

---

## 认证与授权

- [ ] JWT secret 强度足够（至少 32 个字符）
- [ ] JWT secret 存储在环境变量中（非硬编码）
- [ ] Token 过期时间设置合理（访问令牌 15 分钟，刷新令牌 7 天）
- [ ] Passport Strategy 从正确的包导入（passport-jwt，而非 passport-local）
- [ ] JwtModule 的 secret 与 JwtStrategy 的 secretOrKey 匹配
- [ ] 受保护的路由已应用授权守卫
- [ ] Service 中实现了资源级授权
- [ ] 密码哈希使用 bcrypt，cost factor >= 10
- [ ] 刷新令牌安全存储且可撤销

---

## 输入验证

- [ ] 所有请求 DTO 使用了 class-validator 装饰器
- [ ] 全局 ValidationPipe 配置了 whitelist: true
- [ ] 全局 ValidationPipe 配置了 forbidNonWhitelisted: true
- [ ] 启用了输入转换（transform: true）
- [ ] 为复杂业务规则实现了自定义验证器
- [ ] 文件上传已验证（类型、大小、扩展名）
- [ ] 数组输入设置了最大长度限制
- [ ] 字符串输入设置了最大长度限制
- [ ] 数值输入设置了最小/最大范围

---

## 输出清洗

- [ ] 响应中绝不返回密码哈希值
- [ ] 不暴露敏感令牌
- [ ] 用户生成的内容已清洗（XSS 防护）
- [ ] 使用响应 DTO 而非直接返回实体
- [ ] 审计字段（createdAt、updatedAt）不会泄露内部信息
- [ ] 生产环境中错误消息不暴露堆栈跟踪
- [ ] 数据库错误不泄露 Schema 详情

---

## 环境与配置

- [ ] 代码中没有硬编码的 secret
- [ ] 所有 secret 都在环境变量中
- [ ] 启动时验证环境变量
- [ ] .env 文件已加入 .gitignore
- [ ] 开发/预发/生产环境使用不同的配置
- [ ] 数据库 URL 使用了强密码
- [ ] API 密钥定期轮换

## 快速安全扫描命令

```bash
# 检查漏洞
npm audit

# 检查过时的包
npm outdated

# 查找硬编码的 secret（基本检查）
grep -r "password.*=.*['\"]" src/
grep -r "secret.*=.*['\"]" src/
grep -r "apiKey.*=.*['\"]" src/

# 检查 .env 是否被提交
git ls-files | grep .env

# 检查依赖
npm ls --depth=0
```
