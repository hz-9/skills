# prisma migrate status

检查你的数据库迁移状态。

## 命令

```bash
prisma migrate status [options]
```

## 功能说明

- 连接到数据库
- 检查 `_prisma_migrations` 表
- 将已应用的迁移与本地迁移文件进行比较
- 报告：
    - **状态**：数据库是最新的还是落后的
    - **未应用的迁移**：待处理迁移的数量
    - **缺失的迁移**：存在于数据库中但本地缺失的迁移
    - **失败的迁移**：任何应用失败的迁移

## 选项

| 选项 | 描述 |
|--------|-------------|
| `--schema` | schema 文件路径 |
| `--config` | Prisma 配置文件的自定义路径 |

## 示例

### 检查状态

```bash
prisma migrate status
```

输出示例（最新状态）：
```
数据库 schema 是最新的！
```

输出示例（有未应用迁移）：
```
以下迁移尚未应用：
  20240115120000_add_user

要在开发环境中应用迁移，请运行：
  prisma migrate dev

要在生产环境中应用迁移，请运行：
  prisma migrate deploy
```

## 何时使用

- **调试**：为什么 `migrate dev` 抱怨存在漂移？
- **CI/CD**：部署前验证数据库状态
- **生产环境**：检查是否需要迁移（`migrate deploy`）或者部署是否失败

## 退出码

- `0`：成功（可能有待处理的迁移，但命令运行成功）
- `1`：错误

要以编程方式检查待处理的迁移，你可能需要解析输出或使用带有退出码标志的 `migrate diff`。
