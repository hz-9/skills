# NestJS 最佳实践

NestJS 开发的综合最佳实践。

## 核心原则

1. **始终使用构造器注入** - 绝不要使用属性注入
2. **使用 DTO 进行数据传输** - 为请求/响应定义接口
3. **实现正确的错误处理** - 使用异常过滤器
4. **验证所有输入** - 使用验证管道
5. **保持模块专注** - 单一职责原则
6. **使用环境变量** - 绝不硬编码凭据
7. **编写全面的测试** - 单元测试和集成测试
8. **对复杂操作使用事务** - 维护数据一致性
9. **实现正确的日志记录** - 使用拦截器处理横切关注点
10. **使用类型安全** - 充分利用 TypeScript 特性

## 数据库最佳实践

- **连接池** - 在生产环境中始终使用连接池
- **为频繁查询的列建立索引** - 提高查询性能
- **使用迁移进行 schema 更改** - 绝不要手动修改 schema
- **实现软删除** - 保留数据历史
- **使用事务** - 确保数据一致性

## 安全最佳实践

- **绝不暴露堆栈跟踪** - 在生产环境的错误响应中
- **验证所有输入** - 防止注入攻击
- **使用参数化查询** - Drizzle 自动执行此操作
- **实现速率限制** - 防止滥用
- **使用 HTTPS** - 加密传输中的数据
