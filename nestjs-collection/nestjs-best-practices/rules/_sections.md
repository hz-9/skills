# Sections

该文件定义了所有章节、其排序、影响级别和描述。
章节 ID（括号内）是用于对规则进行分组的文件名前缀。

---

## 1. Architecture（架构）(arch)

**影响：** 严重（CRITICAL）
**描述：** 合理的模块组织和依赖管理是维护可维护 NestJS 应用程序的基础。循环依赖和上帝服务是 #1 架构杀手。

## 2. Dependency Injection（依赖注入）(di)

**影响：** 严重（CRITICAL）
**描述：** NestJS 的 IoC 容器功能强大但可能被误用。理解作用域、注入标记和正确模式对于可测试代码至关重要。

## 3. Error Handling（错误处理）(error)

**影响：** 高（HIGH）
**描述：** 一致的错误处理改进了调试、用户体验和 API 可靠性。集中的异常过滤器确保统一的错误响应。

## 4. Security（安全）(security)

**影响：** 高（HIGH）
**描述：** 安全漏洞可能是灾难性的。输入验证、认证、授权和数据保护是必不可少的。

## 5. Performance（性能）(perf)

**影响：** 高（HIGH）
**描述：** 优化请求处理、缓存和数据库查询直接影响应用程序的响应能力和可扩展性。

## 6. Testing（测试）(test)

**影响：** 中-高（MEDIUM-HIGH）
**描述：** 经过良好测试的应用程序更可靠。NestJS 测试工具支持全面的单元和端到端覆盖。

## 7. Database & ORM（数据库与ORM）(db)

**影响：** 中-高（MEDIUM-HIGH）
**描述：** 正确的数据库访问模式、事务处理和查询优化对数据密集型应用程序至关重要。

## 8. API Design（API设计）(api)

**影响：** 中（MEDIUM）
**描述：** RESTful 约定、版本控制、DTO 和一致的响应格式提高了 API 的可用性和可维护性。

## 9. Microservices（微服务）(micro)

**影响：** 中（MEDIUM）
**描述：** 构建分布式系统需要理解消息模式、健康检查和服务间通信。

## 10. DevOps & Deployment（DevOps与部署）(devops)

**影响：** 低-中（LOW-MEDIUM）
**描述：** 配置管理、结构化日志和优雅关闭确保生产就绪和零停机部署。
