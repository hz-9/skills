该仓库作为 AI Agent 技能（Skill）分发中心，其“错误处理”并非传统代码层面的异常捕获机制，而是体现为 **Agent 执行工作流中的状态校验、环境预检、冲突协调及结构化反馈**。核心逻辑通过 Markdown 定义的 SKILL 脚本和 Bash 辅助脚本实现。

### 1. 核心处理模式
- **环境预检与结构化输出**：在执行复杂任务前，通过 `scripts/check-env.sh` 等脚本进行环境探测。脚本统一使用 JSON 格式输出检查结果（包含 `all_passed` 标志位和 `checks` 数组），若检测到关键依赖缺失（如 `jq`）或环境不满足（如非 Git 仓库、存在冲突），则设置 `error` 字段并终止流程。
- **分支与版本控制冲突处理**：针对 Git 操作，定义了明确的冲突解决路径。例如在 `git-branch-prep` 中，若检测到本地落后于远端，自动执行 `rebase`；若产生冲突，则暂停执行并提示用户手动解决。对于受保护分支（Protected Branch），强制要求创建新分支而非直接提交。
- **交互式决策与人工介入（HITL）**：在无法自动化处理的场景下（如多方案选择、疑难 Bug 诊断），采用 `AskUserQuestion` 工具阻塞等待用户决策。对于复杂的 UI 或环境相关 Bug，提供 `hitl-loop.template.sh` 模板，通过“人类在环”的方式捕获错误现场信息。
- **诊断循环与假设验证**：在 `diagnose` 技能中，建立了严格的“反馈循环 → 重现 → 假设 → 检测 → 修复”流程。强调在修复前必须构建确定性的失败信号，并要求对调试日志进行唯一前缀标记以便清理。

### 2. 关键文件与逻辑
- **`skills/git-branch-prep/references/error-handling.md`**：定义了 Git 分支准备过程中的标准错误处理逻辑，包括分支已存在、无变更可分析、Rebase 冲突等场景的应对策略。
- **`skills/*/scripts/check-env.sh`**：多个技能（如 `git-branch-prep`, `changeset-gen`）共用的环境检查脚本，负责输出标准化的 JSON 状态报告。
- **`inbox/skills/diagnose/SKILL.md`**：提供了系统化的 Bug 诊断方法论，涵盖了从构建反馈循环到事后清理的全生命周期错误管理。
- **`skills/git-branch-prep/SKILL.md`**：展示了如何在 Agent 工作流中集成错误检查点（Pre-check），并根据检查结果决定是继续执行、报错终止还是转入异常处理分支。

### 3. 开发者约定
- **前置校验优先**：所有涉及文件系统或版本控制的技能，必须在 Step 0 执行环境预检，确保后续操作的原子性和安全性。
- **错误信息显式化**：脚本错误应通过 JSON 的 `error` 字段明确传达原因（如缺少依赖、权限不足），避免静默失败。
- **禁止裸命令执行**：在执行 `git commit` 等高风险操作时，必须使用环境变量（如 `NO_VERIFY=1`）跳过钩子，防止因外部钩子错误导致流程中断。
- **调试痕迹清理**：在诊断类任务完成后，必须通过标签（如 `[DEBUG-xxx]`）检索并清除所有临时插入的检测代码，确保代码库整洁。
