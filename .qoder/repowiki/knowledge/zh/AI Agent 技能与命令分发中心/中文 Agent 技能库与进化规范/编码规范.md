- 所有涉及用户抉择的交互环节必须使用 `AskUserQuestion` 工具进行阻塞式确认，禁止使用纯文本追问。
- Workflow 步骤必须包含三个 Secure 步骤：前置检查（Step 0）、复核检查（倒数第二步）和成果输出（末尾步骤）。
- 条件分支必须采用树形箭头格式（`是 -> 动作；` / `否 -> 动作；`），且每个分支必须以显式的流程控制指令（如“下一步”或“终止流程”）结尾。
- SKILL.md 文件行数通常限制在 300-500 行以内，超出部分需拆分至 `references/` 目录下，保持引用层次不超过一层。
- 元数据 `description` 必须遵循固定格式：第一句说明能力，第二句说明触发条件（“当...时使用”），并使用第三人称。
