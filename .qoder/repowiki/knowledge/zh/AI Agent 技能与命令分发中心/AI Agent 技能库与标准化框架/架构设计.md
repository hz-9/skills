- **核心规范层**：以 `skill-evolve` 为元技能，定义了技能的“标准模板结构”（8个标准章节）、目录规范（`references/`, `scripts/` 等）及引用层级约束，通过 `template.md` 和 `directory-structure.md` 确立架构基线。
- **生命周期管理层**：`skill-create` 负责从零构建符合规范的技能，`skill-evolve` 负责现有技能的结构对齐、内容精简与引用拆分，`skill-evolve-cycle`  orchestrates 循环优化。
- **领域执行层**：包含 Git 工具链（`git-branch-prep`, `git-cleanup`, `git-commit-helper`）、Monorepo 迁移（`rush-to-nx`）、版本管理（`changeset-gen`）及架构辅助（`zoom-out`, `grill-me`）等具体业务技能。
- **交互与防御机制**：所有技能强制采用 `AskUserQuestion` 进行结构化决策交互，并在 Workflow 中嵌入“预检（Pre-check）”、“审查（Review check）”和“输出（Output）”三个安全步骤，确保执行的原子性与可回溯性。
