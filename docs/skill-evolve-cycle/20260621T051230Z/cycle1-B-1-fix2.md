## 修复问题列表

### 1. 🔴 步骤 0.3-0.4 顺序错误

**问题**：合并检测（原 0.3）在 Git 仓库检查（原 0.4）之前执行，Git 命令可能失败。

**修复**：交换顺序：0.3 = Git 仓库检查 → 0.4 = Git 版本检查 → 0.5 = 合并检测

### 2. 🟡 缺少对话 Diff 路径状态变量

**问题**：Step 6.2 依赖"是否对话 Diff 路径"判断，但无对应状态变量。

**修复**：新增 `是否对话 Diff 路径` 全局变量，添加 Definitions 定义，Step 0.2 中设置。

### 3. 🟡 场景 C 缺少空结果分支

**问题**：`git log <range> -p` 执行成功但返回空结果时无处理。

**修复**：在场景 C 中新增"返回结果为空"分支，引导用户检查范围后重试。

### 4. 🟡 Diff 行数检查绕过对话 Diff 路径

**问题**：2.2 的 diff 行数检查被对话 Diff 路径跳过。

**修复**：将 diff 行数检查移至步骤 3.0（统一检查点），覆盖所有路径。

### 5. 🟡 `git push origin HEAD` 硬编码远程名

**问题**：硬编码 `origin` 不适用于无 origin 或有多 remote 场景。

**修复**：先检测 upstream，失败再 fallback 到 `origin`，都失败则引导用户手动配置。
