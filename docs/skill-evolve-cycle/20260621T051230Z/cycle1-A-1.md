## 发现的问题

### 1. 🔴 标点符号问题（高严重度）

**问题**：中文正文中使用半角直引号 `"` 代替中文弯引号 `""`。

**修复**：将以下位置的半角引号替换为中文弯引号：
- Frontmatter description：`"commit"` → `"commit"`
- Step 0：`"当前不在 Git 仓库中"` → `"当前不在 Git 仓库中"`
- Step 2：`"暂存区"`、`"工作区"`、`"最近 commit"` → 弯引号
- Step 4：`"都不合适"` → `"都不合适"`
- Step 5：`"commit message 已生成完毕"` → `"commit message 已生成完毕"`
- Step 6：`"都不合适"` → `"都不合适"`
- Rules：`"类型"` → `"类型"`
- Rules：`"做了什么"`、`"为什么"` → `"做了什么"`、`"为什么"`
- Definitions：`"都不合适"` → `"都不合适"`
- Review List：`"做了什么"`、`"为什么"` → `"做了什么"`、`"为什么"`

### 2. 🔴 子步骤格式对齐（高严重度）

**问题**：Step 0（前置检查）的子步骤使用项目符号 `-` 格式，而其他步骤（Step 1-6）均使用数字子编号 `N.M` 格式。由于存在跨步骤循环引用（Step 5 返回 Step 4），所有子步骤应统一使用数字子编号格式。

**修复**：将 Step 0 子步骤从 `-` 格式改为 `0.1`-`0.6` 数字子编号格式。

### 3. 🟡 交叉引用格式统一（中严重度）

**问题**：Step 2 中存在简写引用 `进入 2.2`，未按规范使用完整的 `进入步骤 N.M` 格式。

**修复**：将 4 处 `进入 2.2` 均更新为 `进入步骤 2.2`。

### 4. 🟡 子步骤标题加粗格式（中严重度）

**问题**：Step 4 的子步骤（4.1-4.5）使用了加粗标题 `**标题**` 格式，而标准约定数字子编号使用纯文本格式。

**修复**：移除 4.1-4.5 子步骤标题的加粗标记。

### 5. 🟡 Rules 分组重组（中严重度）

**问题**：Rules 使用了 5 个领域特定分组（格式/类型/内容/交互/变更标记），未遵循推荐的 6 维度标准分组（元数据/结构/内容/行为/防御/验证）。

**修复**：经用户确认，重组为 6 维度标准分组中的两个有效维度：
- **内容规范**：合并原有的格式、类型、内容、变更标记相关规则
- **行为规范**：保留原有的交互规范规则
