## 修复问题列表

### 1. 🔴 步骤 0 重组：对话 Diff 优先于 Git 命令

**问题**：步骤 0.2 的 Git MERGE_MSG 检测在 0.3 对话 Diff 路径判断之前执行，当用户处于非 Git 环境的对话 Diff 场景时，Git 命令可能报错。

**修复**：交换步骤顺序：
- 0.2 → 对话 Diff 判断（原 0.3）
- 0.3 → 合并检测（原 0.2）
- 对话 Diff 校验通过时，明确跳过步骤 0.3~0.6

### 2. 🔴 合并冲突检测

**问题**：合并检测仅检查 `MERGE_MSG` 是否存在，未验证冲突是否已解决。

**修复**：在步骤 0.3 中增加 `git diff --name-only --diff-filter=U` 冲突检查。

### 3. 🔴 Push 操作错误处理

**问题**：步骤 6.2 的推送操作仅写"执行推送操作"，无具体命令和错误处理。

**修复**：明确推送命令为 `git push origin HEAD`，增加成功/失败两个分支。

### 4. 🔴 大规模 diff 无保护

**问题**：分支范围和暂存区变更无 diff 大小限制，可能导致 AI 上下文溢出。

**修复**：步骤 2.2 增加 diff 行数检查（2000 行阈值），超限时预警用户。

### 5. 🟡 无效 Git 引用错误处理

**问题**：场景 B 和 C 的 Git 命令无失败处理。

**修复**：为 `git rev-list`、`git log` 增加命令失败分支，引导用户检查后重试。

### 6. 🟡 分支范围语义不明确

**问题**：分支范围作为输入来源时，未说明输出是 squash 式单一 commit message。

**修复**：在步骤 1.1 选项后补充说明"分支范围将作为 squash 式单一 commit message 生成"。

### 7. 🟡 Diff 格式重试超限后行为

**问题**：3 次格式校验失败后行为未定义，AI 行为不一致。

**修复**：在选项下方补充说明：3 次失败后"重新提供"选项不再可用，仅保留"切换到 Git 路径"和"取消"。

### 8. 🟢 "切换到 Git 路径"跳转目标模糊

**问题**："进入 Git 路径检查流程"不是标准术语，跳转目标不明确。

**修复**：改为"跳转到步骤 0.3，进入 Git 路径检查流程"。
