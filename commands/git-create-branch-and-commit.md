## Step 1: 创建分支

如果处于 dev stage staging prod master 分支下，则直接基于当前待提交的内容，创建分支名称；

如果处于 git worktree 模式下的 hash 提交下，并无具体的分支时，
  请查看源于哪个分支，如果是 dev stage staging prod master 分支，则直接基于当前待提交的内容，创建分支名称；
  如果处于某个有效的分支下，则询问使用者，是否创建新的分支，还是在当前分支上进行提交，并推送；

如果处于某个有效的分支下，则询问使用者，是否创建新的分支，还是在当前分支上进行提交，并推送；

## Step 2: 生成提交信息

使用 SKILL: git-commit-helper 进行生成提交信息；请遵守以下规则：

- 采用英文提交信息；
- 不要过度生成内容；
- 不要出现 `[skip ci]` 等跳过 CI 的工具；

## Step3: 进行推送

如果处于 dev stage staging prod master 分支下，则不进行推送；

如果当前提交落后于当前分支远端的最新版本，请 rebase 到最新分支后，再进行推送；

推送完成后，生成一个 PR 的链接输出出来。
