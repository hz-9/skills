# Content Boundary Standard — Define Content Ownership Boundaries Between SKILL.md and references/ Files

## Overview

Define content ownership boundaries between `SKILL.md` and `references/` files. When editing or backporting, query this table to determine where content belongs.

## File Ownership Boundaries

| Content Area                                                                  | Target File                                 |
| ----------------------------------------------------------------------------- | ------------------------------------------- |
| Workflow structure / Secure steps / Step format / Branch logic / Interaction patterns | `references/workflow-standard.md`           |
| Punctuation / Quotes / Brackets / Full-width vs half-width                    | `references/punctuation-convention.md`      |
| Text simplification rules / Safety boundaries                                 | `references/text-optimization.md`           |
| Example format standards / Example consistency rules                          | `references/example-standard.md`            |
| Directory structure & references file specification                           | `references/directory-structure.md`         |
| Rules writing standards                                                       | `references/rules-standard.md`              |
| Review List writing standards                                                 | `references/review-list-standard.md`        |
| Variable declaration standards                                                | `references/rules-standard.md`              |
| Metadata / Structure / Content / Behavior / Defense / Validation standards    | Remain in SKILL.md Rules                    |
