# Grill Me

## Overview

Continuously question every aspect of the user's plan or design, following each [decision tree branch](#decision-tree-branch) to completion, resolving dependencies between decisions one by one. Provide a recommended answer for each question, ask one question at a time, until consensus is reached.

## Definitions

- <a id="grill"></a>**Grill**: Systematically challenge and question every decision point of a plan, exposing potential risks and oversights.
- <a id="decision-tree-branch"></a>**Decision Tree Branch**: Interdependent decision paths within a plan, each branch must be independently traversed to completion.

## Prerequisites

- The user has a plan, design, or proposal ready to be grilled.
- The Agent can browse the project codebase to verify technical questions, instead of asking the user.

## Workflow

0. **Pre-flight Check** — Ensure grilling conditions are met;
    - Check if the user has a plan or proposal ready to be grilled:
        - Yes -> enter Step 1 (Understand Plan);
        - No -> provide options via AskUserQuestion, block and wait for user selection:
          - Describe the plan yourself -> wait for user to input plan content, after receiving return to Step 0 and re-check;
          - AI-guided -> AI guides the user step by step to articulate the plan, after completion return to Step 0 and re-check;

1. **Understand Plan** — Identify core decision points;
    - Prioritize branches with the most dependencies and highest risk as the starting branch;
    - After understanding is complete -> enter Step 2 (Start Grilling);

2. **Start Grilling** — Dive deeper along [decision tree branches](#decision-tree-branch);
    - Check if the current question can be answered by browsing the codebase:
        - Yes -> verify directly instead of asking the user, then check if there are still questions to ask:
          - Still questions -> continue with subsequent steps in this workflow;
          - All questions resolved -> jump to Step 7 (Output Results);
        - No -> proceed to the next step;
    - Provide recommended answer and options via AskUserQuestion, block and wait for user selection (ask one question at a time):
      - Agree with recommended answer -> record decision, enter consensus judgment;
      - Disagree -> ask for specific reasons via AskUserQuestion, provide alternatives, block and wait for user selection:
        - [Dynamic options generated by AI based on user feedback] -> record decision, enter consensus judgment;
    - **Consensus Judgment** — Check if consensus has been reached on the current branch:
        - Yes -> next step;
        - No -> record disagreements and continue subsequent questioning to avoid getting stuck;

3. **Scope Check** — Determine if the current line of questioning is out of scope;
    - Check if the question involves infrastructure or underlying technology choices that do not affect plan feasibility:
        - Yes -> provide options via AskUserQuestion, block and wait for user selection:
          - Continue deeper -> continue questioning, proceed to next step;
          - Skip branch -> skip current branch, proceed to next step;
        - No -> next step;
    - Check if the questioning deviates from the core goal by more than two layers of dependency:
        - Yes -> provide options via AskUserQuestion, block and wait for user selection:
          - Continue deeper -> continue questioning, proceed to next step;
          - Return to mainline -> return to core goal, proceed to next step;
        - No -> next step;

4. **Backtrack Check** — Determine if any conflicts with previous decisions are discovered;
    - Check if conflicts with previous decisions are discovered:
      - Yes -> backtrack to related branches and renegotiate:
        - After backtracking, check if previously recorded disagreement is encountered again:
          - Yes -> escalate the disagreement to a blocker, output summary and suggest separate discussion; terminate flow;
          - No -> continue questioning;
      - No -> next step;

5. **Traversal Check** — Determine if all decision tree branches have been traversed;
    - Check if all decision tree branches have been traversed:
      - Yes -> next step;
      - No -> return to Step 2, select the next decision branch;

6. **Review Check** — Check against [Review List](#review-list) to confirm execution results;
    - Check if Review List has content:
        - No -> go directly to next step (output results);
        - Yes -> next step;
    - Check each item in [Review List](#review-list) in order, check if passed:
        - Display output content based on "Review Check Example";
        - Any check fails -> terminate flow;
        - All pass -> next step;

7. **Output Results** — Output decision summary;
    - Check if consensus was reached:
        - Yes -> output the final decision summary;
        - No (user terminated) -> output the summary of consensus reached and unresolved disagreements;

## Rules

### Questioning Conventions
    - Ask only one question at a time to avoid information overload.
    - Each question must provide a recommended answer.
### Decision Conventions
    - Questions that can be answered by browsing the codebase -> verify directly, do not ask the user.
    - Traverse all decision tree branches, do not miss any. Cannot make decisions on your own, must obtain user consent.
### Disagreement Handling
    - When the user disagrees with the recommended answer, ask for specific reasons and provide alternatives.
    - If consensus still cannot be reached, record the disagreement and continue subsequent questioning to avoid getting stuck.
### Backtrack Conventions
    - When conflicts with previous decisions are discovered, must backtrack to related branches and renegotiate until all branch decisions remain logically consistent.
    - If a previously recorded disagreement is encountered again during backtracking, escalate the disagreement to a blocker, output the summary and suggest separate discussion to avoid infinite loops.
### Scope Conventions
    - Grilling scope should be limited to the direct decision-making level of the user's plan. Do not question infrastructure or underlying technology choices unless the reason directly affects plan feasibility.
    - If questioning deviates from the core goal by more than two layers of dependency, remind the user whether to continue deeper.
### Termination Conventions
    - Stop questioning once consensus is reached, output the final decision summary.
    - When the user actively terminates, output the summary of consensus reached and unresolved disagreement points.
### Interaction Conventions
    - All user decision-making interactions **must** use the `AskUserQuestion` tool; plain text follow-ups are prohibited; pass questions and options into AskUserQuestion in a structured way, no more than 4 questions per call;
### Self-Consistency Conventions
    - Review List check items must cover all binding rules in Rules, ensuring one-to-one correspondence between Rules and Review List;

## Review List

- **Questioning Quality Check**
    - [ ] Only one question asked at a time, no question bombing
    - [ ] Each question provided a recommended answer
- **Completeness Check**
    - [ ] Questions answerable via codebase have been verified rather than asked
    - [ ] All decision tree branches traversed, none missed
    - [ ] No decisions made autonomously, each decision obtained user consent
    - [ ] Decision conflicts discovered and backtracked for renegotiation
