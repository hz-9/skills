# 深层模块（Deep Modules）

源自《软件设计哲学》（A Philosophy of Software Design）：

**深层模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   小型接口          │  ← 少量方法，简单参数
├─────────────────────┤
│                     │
│                     │
│   深层实现          │  ← 复杂逻辑被隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅层模块** = 大接口 + 少量实现（避免）

```
┌─────────────────────────────────┐
│       大型接口                   │  ← 大量方法，复杂参数
├─────────────────────────────────┤
│  薄的实现                        │  ← 仅仅是透传
└─────────────────────────────────┘
```

在设计接口时，问自己：

- 我能减少方法的数量吗？
- 我能简化参数吗？
- 我能在内部隐藏更多复杂性吗？
