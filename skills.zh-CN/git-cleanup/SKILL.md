# Git Cleanup

## Overview

系统化清理 Git 仓库中废弃的分支和 Tag。按“扫描 -> 展示 -> 用户确认 -> 删除”流程操作，先处理本地分支和 Tag，再经二次确认后统一删除远程引用。执行前自动创建备份。

## Definitions

- <a id="废弃分支"></a>**废弃分支**：符合以下任一条件的分支：
    - **已合并分支**：已被合并到主分支的本地分支；
    - **孤儿分支**：执行 `git fetch -p` 后，远程跟踪分支已不存在的本地分支（显示 `: gone]`）；
- <a id="废弃 Tag"></a>**废弃 Tag**：符合以下任一条件的 Tag：
    - **非版本 Tag**：不匹配语义化版本号模式（如 `v1.2.3`、`1.2.3`、`v1.2.3-beta.1`）的 Tag；
    - **孤儿 Tag**：指向的 commit 没有任何本地或远程分支引用的 Tag；
- <a id="主分支"></a>**主分支**：仓库的主要开发分支（如 `main` 或 `master`），由 AI 自动检测后向用户确认；
- <a id="保护分支"></a>**保护分支**：主分支及 `develop` 分支，自动跳过、不可删除；
- <a id="当前检出分支"></a>**当前检出分支**：`HEAD` 指向的分支，自动跳过、不可删除。

## Prerequisites

- Git 2.0+；
- 当前在 Git 仓库目录中（可通过 `git rev-parse --git-dir` 验证）；
- 用户有对远程仓库的推送权限（如需执行远程删除）。

## Workflow

0. **前置检查** — 验证环境、检测主分支、创建备份；
    - 验证当前是否在 Git 仓库中：
        - 是 -> 下一步；
        - 否 -> 报告“当前不在 Git 仓库中”，终止流程；
    - 自动检测主分支（依次检测 `refs/heads/main`、`refs/heads/master`、`refs/heads/develop`）：
        - 检测到 -> 通过 AskUserQuestion 提供选项，阻塞等待用户选择：
            - 确认主分支名称 -> 使用该名称，执行后进入下一步；
            - 手动指定其他分支 -> 由用户输入分支名称，执行后进入下一步；
        - 未检测到 -> 通过 AskUserQuestion 提供选项，阻塞等待用户选择：
            - 手动指定主分支名称 -> 由用户输入分支名称，执行后进入下一步；
    - 识别[当前检出分支](#当前检出分支)（`git branch --show-current`），存入内存标记为跳过；
    - 创建备份（`cp -a` 在仓库同级目录，命名格式 `<仓库名>.bak.<YYYYMMDDTHHMMSS>`）：
        - 成功 -> 下一步；
        - 失败 -> 报告“备份创建失败”，通过 AskUserQuestion 提供选项，阻塞等待用户选择：
            - 跳过备份继续 -> 进入步骤 1；
            - 终止 -> 终止流程；
    - 执行 `git fetch -p` 修剪已不存在的远程跟踪引用；

1. **扫描并删除废弃 Branch（本地）** — 收集并展示待清理分支；
    - 扫描已合并分支：`git branch --merged <主分支> --format='%(refname:short)'`，过滤掉[保护分支](#保护分支)和[当前检出分支](#当前检出分支)；
    - 扫描孤儿分支：`git branch -vv | awk '/: gone]/{print $1}'`，过滤掉[当前检出分支](#当前检出分支)；
    - 合并两份列表为候选集合，去重；
    - 判断候选集合是否为空：
        - 是 -> 报告“未发现待清理的废弃分支”，进入步骤 2；
        - 否 -> 以表格形式展示（列：分支名 / 类型 / 原因），通过 AskUserQuestion 提供选项，阻塞等待用户选择：
            - 确认删除全部 -> 删除全部候选分支，执行后进入下一步；
            - 选择部分删除 -> 由用户指定要删除的分支，执行后进入下一步；
            - 跳过全部 -> 进入步骤 2；
    - 对用户确认删除的每个分支，执行删除：
        - 已合并分支：`git branch -d <分支名>`；
        - 孤儿分支：`git branch -D <分支名>`（因孤儿分支无远程跟踪，`-d` 可能拒绝删除，需使用 `-D`）；
        - 记录每条删除结果（成功 / 失败及原因）；

2. **扫描并删除废弃 Tag（本地）** — 收集并展示待清理 Tag；
    - 扫描非版本 Tag：`git tag -l | grep -v -E '^v?[0-9]+\.[0-9]+\.[0-9]+'`；
    - 扫描孤儿 Tag：遍历所有 Tag，对每个 Tag 执行 `git branch --all --contains <Tag名>`：
        - 输出为空 -> 标记为孤儿 Tag；
        - 输出非空 -> 跳过；
    - 合并两份列表为候选集合，去重；
    - 判断候选集合是否为空：
        - 是 -> 报告“未发现待清理的废弃 Tag”，进入步骤 3；
        - 否 -> 以表格形式展示（列：Tag 名 / 类型 / 原因），通过 AskUserQuestion 提供选项，阻塞等待用户选择：
            - 确认删除全部 -> 删除全部候选 Tag，执行后进入下一步；
            - 选择部分删除 -> 由用户指定要删除的 Tag，执行后进入下一步；
            - 跳过全部 -> 进入步骤 3；
    - 对用户确认删除的每个 Tag，执行 `git tag -d <Tag名>`，记录每条删除结果；

3. **远程删除（二次确认）** — 输出摘要后统一删除远程引用；
    - 输出本地删除摘要表格：
        - 分支：列出已删除的分支名和数量，以及失败项（如有）；
        - Tag：列出已删除的 Tag 名和数量，以及失败项（如有）；
        - 自动跳过：列出[当前检出分支](#当前检出分支)、[保护分支](#保护分支)；
    - 判断本次是否有已成功本地删除的废弃引用：
        - 是 -> 通过 AskUserQuestion 提供选项，阻塞等待用户选择：
            - 确认推送远程删除 -> 执行远程删除，执行后进入下一步；
            - 仅保留本地删除 -> 跳过远程删除，执行后进入下一步；
            - 查看详情后再决定 -> 展示详细删除列表，再次通过 AskUserQuestion 确认；
        - 否（本地无任何删除）-> 报告“无变更需要同步到远程”，进入步骤 4；
    - 对确认远程删除的引用，批量执行：
        - 删除远程分支：`git push origin --delete <分支名1> <分支名2> ...`；
        - 删除远程 Tag：`git push origin --delete refs/tags/<Tag名1> refs/tags/<Tag名2> ...`；
    - 输出最终清理报告（本地 + 远程删除统计）。

4. **复核检查** — 对照 [Review List](#review-list)，确认执行结果；
    - 判断 Review List 是否有内容：
        - 否 -> 直接进入下一步（成果输出）；
        - 是 -> 下一步；
    - 依次判断 [Review List](#review-list) 中每个检查项，是否通过（基于“复核检查示例”显示输出内容）：
        - 是 -> 继续下一个检查项；
        - 否 -> 记录失败检查项，终止流程；
    - 全部通过后，进入下一步（成果输出）；

5. **成果输出** — 输出清理摘要，告知完成；
    - 输出结构化摘要（包含本次清理的各维度统计数量）；
    - 告知清理完成；

## Rules

- **元数据规范**
    - description 遵循格式：第一句说明技能能做什么，第二句说明触发条件（“当...时使用”），使用第三人称，不超过 1024 字符；
- **结构规范**
    - 标准目录仅适用于 SKILL.md 本身，不影响目录下其他文件；
- **内容规范**
    - 扫描结果和删除摘要均以表格形式呈现；
    - 表格列名清晰，包含“引用名 / 类型 / 原因 / 状态”等必要维度；
    - 最终清理报告包含各维度的统计数量（已删除 / 已跳过 / 失败等）；
- **行为规范**
    - 执行任何删除操作前必须已完成备份（`cp -a`）；
    - [当前检出分支](#当前检出分支)自动跳过，不进入扫描列表；
    - [保护分支](#保护分支)（[主分支](#主分支) + `develop`）自动跳过，不进入扫描列表；
    - 远程删除必须经过二次确认，禁止在首次确认后直接推送删除；
    - 所有涉及用户抉择的交互环节（确认删除、选择操作方式、确认[主分支](#主分支)等），**必须**使用 `AskUserQuestion` 工具，禁止以纯文本追问代替；将问题与选项结构化传入 AskUserQuestion，每次调用不超过 4 个问题；
    - 先处理 Branch，再处理 Tag；
    - 先删除本地引用，再删除远程引用（分两阶段）；
    - 本地删除完成后必须输出删除摘要，再进行二次确认；
    - 删除命令执行失败时，记录失败原因并继续执行后续删除，不中断流程；
- **防御规范**
    - 删除任何文件，都要通过交互式提问向用户进行确认；
- **验证规范**
    - 示例内容应与最新 Workflow 步骤一致；

## Review List

- **内容检查**
    - [ ] 所有用户确认删除的分支已成功删除（或已记录失败原因）
    - [ ] 所有用户确认删除的 Tag 已成功删除（或已记录失败原因）
    - [ ] 远程删除命令已正确执行，无推送错误
    - [ ] 扫描结果以表格形式展示，列名清晰
    - [ ] 删除摘要包含各维度统计（已删除 / 已跳过 / 失败）
    - [ ] 最终清理报告包含本地和远程两个维度的统计
- **行为检查**
    - [ ] 执行前已创建 `cp -a` 备份，路径和命名正确
    - [ ] [当前检出分支](#当前检出分支)未被列入删除列表
    - [ ] [保护分支](#保护分支)（[主分支](#主分支) + develop）未被列入删除列表
    - [ ] 远程删除前经过二次确认
    - [ ] 遵循了“先 Branch 后 Tag”的执行顺序
    - [ ] 遵循了“先本地后远程”的两阶段删除流程
- **防御检查**
    - [ ] 删除命令执行失败时，已正确记录失败原因并继续执行后续删除
- **验证检查**
    - [ ] 无中断迹象：无未完成的删除操作
