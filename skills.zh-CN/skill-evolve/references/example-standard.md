# 示例编写规范 — 定义 SKILL.md 中 `## Examples` 节的编写格式与自洽性规则

## Overview

规范 SKILL.md 中 `## Examples` 节的编写格式与内容要求，涵盖对话交互示例、复核检查示例、成果输出示例三种类型的编写标准和自洽性规则。

## 对话交互示例规范

- 展示用户与 AI 的对话流程，帮助理解 SKILL 的触发方式和 AI 的响应行为
- 每个示例由一条加粗标题（`**标题**`）概括场景
- 紧跟一个 ````markdown 代码块包裹对话内容
- 多个示例间空行分隔

## 复核检查示例规范

- 展示复核检查执行时的输出格式和样式，供 AI 参考输出结构
- 示例中可按组展示**代表性通过项**（每组 1-2 项），完整展示失败场景；末尾标注 “（AI 运行时将逐项输出全部检查项结果）”
- AI 运行时必须逐项输出 Review List **全部**检查项结果，不得以“其余项已通过”等缩写形式跳过
- 使用 🟩 标注通过项、🟥 标注失败项
- 检查失败时必须显式终止流程

## 成果输出示例规范

- 展示 SKILL 执行后的成果状态，让用户直观理解应用效果
- 每个示例由一条加粗标题（`**标题**`）概括场景
- 紧跟一个 ````markdown 代码块包裹结果表格
- 多个示例间空行分隔

## 示例自洽性规则

- Examples 内容必须自洽于 Rules（示例中的行为、流程、数值须与 Rules 定义一致）
- Examples 必须包含复核检查示例（展示 Review List 验收失败的终止流程）
- 对话交互示例仅聚焦步骤 0-5（不含复查和输出等 Secure 步骤）
- `## Examples` 中的所有示例已由 ````markdown 代码块包裹
- 修改或重命名 Workflow 步骤后，须同步检查对话交互示例中引用的步骤名
- 示例流程展示应与最新 Workflow 流程一致
- 示例中的数字（步骤数、检查项数、行数等）须与实际值同步
- Examples 中的数字应使用与文件实际状态**解耦**的通用示例值，避免使用文件行数、检查项数等易变数字；
- 若确需展示与实际值相关的数字（如“38 个检查项”），须在步骤3的易变数字扫描中同步更新；
