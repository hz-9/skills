# 标点符号使用规范 — 规范 SKILL.md 及 references/ 下文件的标点符号使用

## 概述

规范 SKILL.md 及 references/ 下文件中标点符号的使用，确保全角/半角使用一致、分支标记符号统一、引号/括号遵循语言规范。避免因符号不一致导致 AI 格式学习紊乱。

## 核心原则

1. **语言决定符号**：中文内容使用全角标点，英文内容使用半角标点（仅适用于正文；代码块、内联代码、文件路径始终使用半角）。
2. **分支符号绑定**：分支逻辑的树形箭头格式绑定特定符号（`；` 为终止符，`：` 为子操作引入符），不因语言切换改变。
3. **规范文件自指一致性**：所有 reference 文件自身的标点使用必须符合本文定义，不得出现内部不一致。

## 规则 1：全角/半角区分

### 1.1 中文正文

- 句号 `。`、逗号 `，`、冒号 `：`、分号 `；`、括号 `（）`、引号 `""` `''` — 均为全角
- 破折号：统一使用 `—`（前后各空一格的全角破折号）
- 书名号：`《》`

### 1.2 英文正文

- 句号 `.`、逗号 `,`、冒号 `:`、分号 `;`、括号 `()`、引号 `"` `'` — 均为半角

### 1.3 混合内容

- 代码块、内联代码（`` `code` ``）、文件路径（`path/to/file`）始终使用半角，不因上下文语言改变
- 技术术语中的半角符号（如 `v2.1.0`、`README.md`）保持半角
- 全角/半角不混用：不要在中文句中使用半角逗号，也不要在英文句中使用全角逗号

### 示例

```markdown
**正确**（中文使用全角）
请确认 `name` 是否存在：若缺失，通过 AskUserQuestion 提供选项。

**错误**（中文中使用半角冒号和逗号）
请确认 `name` 是否存在: 若缺失, 通过 AskUserQuestion 提供选项。
```

## 规则 2：分支逻辑标记符号

与 [Workflow 编写规范](workflow-standard.md#分支逻辑写作规范) 中的分支逻辑标记符号保持一致：

| 符号             | 位置                              | 含义                                   |
| ---------------- | --------------------------------- | -------------------------------------- |
| `；`（全角分号） | 每个分支动作行末                  | 分支结束标记，表示此路径终止           |
| `：`（全角冒号） | 引入子条件/子操作的非终止分支行末 | 子操作展开标记                         |
| `->`（半角箭头） | 分支条件与动作之间                | 分支走向指示，**禁止使用全角箭头 `→`** |

**禁止**：

- 使用 `;` 替代 `；`（半角分号替代全角分号不可接受，因为视觉差异可能导致 AI 忽略规则）
- 在非分支逻辑的普通文本中使用 `；` 作为列举分隔符（应使用 `、` 或 `，`）
- 使用全角箭头 `→` 替代 `->`；所有箭头符号必须使用半角 `->`，包括文本描述中的“A → B”写法也应替换为“A -> B”
- 在动作与流程控制（如 `下一步；`）之间使用 `；` 作为分隔符；当动作后需显式标注流程走向时，应使用 `，` 连接，如 `执行动作，执行后进入下一步；`

### 示例

```markdown
**正确**

- 判断文件是否存在：
  - 是 -> 读取内容；
  - 否 -> 终止流程；

**错误**（半角分号代替全角分号）

- 判断文件是否存在：
  - 是 -> 读取内容;
  - 否 -> 终止流程;
```

## 规则 3：引号使用

| 语言           | 引号类型         | 示例                      |
| -------------- | ---------------- | ------------------------- |
| 中文           | 弯引号 `""` `''` | 报告“目标不存在”          |
| 中文引用内引用 | `'...'`          | “他说'好的'”              |
| 英文           | 直引号 `"` `'`   | Report "target not found" |
| 代码内联       | 反引号 `` ` ``   | `name` 字段               |

**禁止**：

- 中文中使用直引号（代码上下文以外，中文正文的引号应使用弯引号而非半角直引号）
  - 错误示例（直引号）：`”目标不存在”`
  - 正确示例（弯引号）：`\u201c目标不存在\u201d` 或使用 editor 的智能引号功能键入
  - 说明：弯引号（也称 Curly Quotes）在中文排版中呈现为 `\u201c...\u201d`（左/右不对称），与直引号 `""`（左右对称）不同。若 editor 无法自动转换，可手动输入 Unicode 码点或使用支持智能引号的编辑工具。
- 英文中使用弯引号

## 规则 4：括号使用

| 语言      | 括号类型    | 示例                                       |
| --------- | ----------- | ------------------------------------------ |
| 中文      | 全角 `（）` | 覆盖（覆盖文件） / 合并（合并内容）       |
| 英文      | 半角 `()`   | Provide options (overwrite / merge / skip) |
| 代码/路径 | 半角 `()`   | 仅用于函数调用或技术语法                   |

**注意**：`AskUserQuestion` 的选项列表已改为缩进子列表格式，不再使用括号包裹选项。括号仅用于非 AskUserQuestion 场景的内联枚举。

## 规则 5：省略号与列举

- 列举三要素用 `、`（中文）或 `, `（英文）：`覆盖、合并、跳过` 或 `overwrite, merge, skip`
- `...`（三个半角点）仅用于代码或技术上下文，不做中文省略号使用
- 中文省略号应为 `……`（两个全角点），但在 SKILL.md 中应避免不必要的省略号

## 规则 6：特殊符号

### 6.1 双向箭头 `↔`

`↔` 表示双向映射关系，仅在 Definitions 中用于展示术语间的等价关系。

> **示例**：`Description`↔`Overview`（表示二者含义相同，可互相映射）

**禁止**：

- 在 Workflow 中使用 `↔` 替代流程方向（流程方向应使用 `->` 半角箭头）
- 在 Rules 或 Review List 中使用 `↔`

### 6.2 全角破折号 `—`

- 全角破折号 `—` 仅在标题/条目中的分隔符场景使用，前后各空一格：`步骤名称 — 步骤说明`
- 禁止在正文内使用全角破折号代替逗号或句号
- 禁止使用两个及以上连续全角破折号 `———`

### 6.3 波浪号 `～`

- 在 SKILL.md 中避免使用波浪号 `～`
- 表示范围时使用文字表述（如”3 到 5 个”），不使用 `3～5`

### 6.4 反引号

- 反引号（`` ` ``）仅用于包裹内联代码、文件名、文件路径、工具名称
- 禁止在非代码上下文中使用反引号替代引号

## 常见反模式

### 反模式 1：全角/半角混用

> **错误**：`通过 AskUserQuestion 提供选项(覆盖/合并/跳过)，阻塞等待用户选择`
> **问题**：括号为半角但中文正文使用全角逗号，风格不一致
> **正确**：
>
> ```markdown
> 通过 AskUserQuestion 提供选项，阻塞等待用户选择：
>     - 覆盖 -> 下一步；
>     - 合并 -> 下一步；
>     - 跳过 -> 下一步；
> ```

### 反模式 2：分号误用

> **错误**：`下一步；继续检查下一个文件；`
> **问题**：同一行内使用两个全角分号，违反了“每个分号是单条分支的结束标记“
> **正确**：拆分为两行或用逗号连接非分支内容

### 反模式 3：语言切换不彻底

> **错误**：英文 SKILL.md 中出现 `下一步；`（中文符号）
> **问题**：同步翻译时未更新标点
> **正确**：英文用 `Next step;`

### 反模式 4：分号分隔动作与流程控制

> **错误**：`覆盖文件；下一步；`
> **问题**：同一行内堆叠两个全角分号，第一个 `；` 被理解为分支结束标记，但后续又出现 `下一步；`，造成语义冲突
> **正确**：`覆盖文件，执行后进入下一步；`（使用逗号连接动作与流程控制）

## 验证清单

- [ ] 中文内容使用全角标点（`。，：；（）”"`），英文内容使用半角标点（`.,:;()"`）
- [ ] 代码块、内联代码、文件路径使用半角符号
- [ ] 分支逻辑：终止行以 `；` 结尾，非终止行以 `：` 结尾
- [ ] 引号：中文用弯引号 `""`，英文用直引号 `"`
- [ ] 括号：中文用全角 `（）`，英文用半角 `()`
- [ ] 无全角/半角混用（同一段落的同类型符号一致）
- [ ] 省略号：中文正文使用 `……`（两个全角点），`...` 仅用于代码上下文
- [ ] 双向箭头 `↔` 仅在 Definitions 中使用，不在 Workflow/Rules/Review List 中出现
- [ ] 全角破折号 `—` 仅作为标题分隔符使用（前后各空一格），不连续使用
- [ ] 规范性文件自身通过上述所有检查项
