# 标点符号使用规范

## 概述

规范 SKILL.md 及 references/ 下文件中标点符号的使用，确保全角/半角使用一致、分支标记符号统一、引号/括号遵循语言规范。避免因符号不一致导致 AI 格式学习紊乱。

## 核心原则

1. **语言决定符号**：中文内容使用全角标点，英文内容使用半角标点（仅适用于正文；代码块、内联代码、文件路径始终使用半角）。
2. **分支符号绑定**：分支逻辑的树形箭头格式绑定特定符号（`；` 为终止符，`：` 为子操作引入符），不因语言切换改变。
3. **规范文件自指一致性**：所有 reference 文件自身的标点使用必须符合本文定义，不得出现内部不一致。

## 规则 1：全角/半角区分

### 1.1 中文正文

- 句号 `。`、逗号 `，`、冒号 `：`、分号 `；`、括号 `（）`、引号 `""` `''`——均为全角
- 破折号：统一使用 ` — `（前后空格的全角破折号）
- 书名号：`《》`

### 1.2 英文正文

- 句号 `.`、逗号 `,`、冒号 `:`、分号 `;`、括号 `()`、引号 `"` `'`——均为半角

### 1.3 混合内容

- 代码块、内联代码（`` `code` ``）、文件路径（`path/to/file`）始终使用半角，不因上下文语言改变
- 技术术语中的半角符号（如 `v2.1.0`、`README.md`）保持半角
- 全角/半角不混用：不要在中文句中使用半角逗号，也不要在英文句中使用全角逗号

### 示例

```markdown
**正确**（中文使用全角）
请确认 `name` 是否存在：若缺失，通过 AskUserQuestion 提供选项。

**错误**（中文中使用半角冒号和逗号）
请确认 `name` 是否存在: 若缺失, 通过 AskUserQuestion 提供选项。
```

## 规则 2：分支逻辑标记符号

与 [分支逻辑写作规范](branch-logic-writing.md) 保持一致：

| 符号 | 位置 | 含义 |
|------|------|------|
| `；`（全角分号） | 每个分支动作行末 | 分支结束标记，表示此路径终止 |
| `：`（全角冒号） | 引入子条件/子操作的非终止分支行末 | 子操作展开标记 |
| `->`（半角箭头） | 分支条件与动作之间 | 分支走向指示，**禁止使用全角箭头 `→`** |

**禁止**：
- 使用 `;` 替代 `；`（半角分号替代全角分号不可接受，因为视觉差异可能导致 AI 忽略规则）
- 在非分支逻辑的普通文本中使用 `；` 作为列举分隔符（应使用 `、` 或 `，`）
- 使用全角箭头 `→` 替代 `->`；所有箭头符号必须使用半角 `->`，包括文本描述中的"A → B"写法也应替换为"A -> B"

### 示例

```markdown
**正确**
- 判断文件是否存在：
    - 是 -> 读取内容；
    - 否 -> 终止流程；

**错误**（半角分号代替全角分号）
- 判断文件是否存在：
    - 是 -> 读取内容;
    - 否 -> 终止流程;
```

## 规则 3：引号使用

| 语言 | 引号类型 | 示例 |
|------|----------|------|
| 中文 | 弯引号 `""` `''` | 报告"目标不存在" |
| 中文引用内引用 | `'...'` | "他说'好的'" |
| 英文 | 直引号 `"` `'` | Report "target not found" |
| 代码内联 | 反引号 `` ` `` | `name` 字段 |

**禁止**：
- 中文中使用直引号（`"目标不存在"` 错误，应为 `"目标不存在"`）
- 英文中使用弯引号

## 规则 4：括号使用

| 语言 | 括号类型 | 示例 |
|------|----------|------|
| 中文 | 全角 `（）` | 提供选项（覆盖 / 合并 / 跳过） |
| 英文 | 半角 `()` | Provide options (overwrite / merge / skip) |
| 代码/路径 | 半角 `()` | 仅用于函数调用或技术语法 |

**注意**：`AskUserQuestion` 的选项列表括号跟随 SKILL.md 的主语言。中文 SKILL.md 中使用 `（选项1 / 选项2）`，英文版使用 `(option1 / option2)`。

## 规则 5：省略号与列举

- 列举三要素用 `、`（中文）或 `, `（英文）：`覆盖、合并、跳过` 或 `overwrite, merge, skip`
- `...`（三个半角点）仅用于代码或技术上下文，不做中文省略号使用
- 中文省略号应为 `……`（两个全角点），但在 SKILL.md 中应避免不必要的省略号

## 常见反模式

### 反模式 1：全角/半角混用

> **错误**：`通过 AskUserQuestion 提供选项(覆盖/合并/跳过)，阻塞等待用户选择`
> **问题**：括号为半角但中文正文使用全角逗号，风格不一致
> **正确**：`通过 AskUserQuestion 提供选项（覆盖 / 合并 / 跳过），阻塞等待用户选择`

### 反模式 2：分号误用

> **错误**：`下一步；继续检查下一个文件；`
> **问题**：同一行内使用两个全角分号，违反了"每个分号是单条分支的结束标记"
> **正确**：拆分为两行或用逗号连接非分支内容

### 反模式 3：语言切换不彻底

> **错误**：英文 SKILL.md 中出现 `下一步；`（中文符号）
> **问题**：同步翻译时未更新标点
> **正确**：英文用 `Next step;`

## 验证清单

- [ ] 中文内容使用全角标点（`。，：；（）""`），英文内容使用半角标点（`.,:;()"`）
- [ ] 代码块、内联代码、文件路径使用半角符号
- [ ] 分支逻辑：终止行以 `；` 结尾，非终止行以 `：` 结尾
- [ ] 引号：中文用弯引号 `""`，英文用直引号 `"`
- [ ] 括号：中文用全角 `（）`，英文用半角 `()`
- [ ] 无全角/半角混用（同一段落的同类型符号一致）
- [ ] 规范文件自身通过上述所有检查项
