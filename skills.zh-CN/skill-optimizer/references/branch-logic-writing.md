# 分支逻辑写作规范

## 概述

规范 SKILL.md 中条件分支判断的写法，将平铺的 if-else 转为树形箭头格式（`是 → / 否 →`），确保每种分支结果对应明确、可执行的操作指令。

## 规则 1：分支逻辑树形化

条件分支必须以树形箭头格式呈现，每种结果（是/否）独占一行、缩进表达层级，避免将多个分支结果压缩在同一行。

> **标准格式**：
> ```markdown
> - 判断条件；
>     - 是 -> 执行动作 A；
>     - 否 -> 执行动作 B；
> ```

### 示例 1：简单二元判断

```markdown
**原文**

- 若缺失或不规范，交互式确认修复方案；

**优化后**

- 确认 `name` 是否存在；
    - 是 -> 下一步；
    - 否 -> 通过 AskUserQuestion 提供几个选项（文件夹名，基于 SKILL.md 整理出来的文件名），阻塞等待用户选择；
```

> **关键改进**：原文只有"若缺失"的隐含分支，AI 不知道满足条件时该做什么；优化后明确定义了"是→下一步"。

### 示例 2：嵌套分支

```markdown
**原文**

- 非模板标准目录的内容 → 迁移到 `references/` 下（若存在同名文件，交互式确认覆盖、合并或跳过）；

**优化后**

- 非模板标准目录的内容 → 迁移到 `references/` 下
    - 是否存在同名文件:
        - 是 -> 通过 AskUserQuestion 提供选项（覆盖 / 合并 / 跳过），阻塞等待用户选择；
        - 否 -> 下一步；
```

> **关键改进**：原文将"迁移"和"同名文件处理"压缩在一行，AI 可能混淆执行顺序；优化后通过缩进表达了父子判断关系。

### 示例 3：存在性检测

```markdown
**原文**

- 若存在 `REFERENCE.md`，先展示拆分方案，确认后执行：

**优化后**

- 检测 `REFERENCE.md` 是否存在：
    - 是 -> 通过 AskUserQuestion 提供拆分选项（由 AI 根据 REFERENCE.md 内容生成），阻塞等待用户选择；
    - 否 -> 下一步
```

> **关键改进**：原文缺少"不存在时"的行为定义；优化后补充了 `否 → 下一步`，消除 AI 的行为盲区。

## 危险场景分析

> **场景 1：缺少否定分支**
> *示例*：`若文件存在则读取内容` → 缺少"不存在时"的行为
> *问题*：AI 遇到文件不存在时可能报错或跳过，行为不可预测
> *正确*：明确 `否 -> 提示用户文件不存在并退出`
>
> **场景 2：多分支压缩**
> *示例*：`若 A 则 X，若 B 则 Y` → 两个独立条件写在同一行
> *问题*：AI 难以判断 A 和 B 是否互斥，可能同时执行 X 和 Y
> *正确*：拆分为两个独立的树形判断块
>
> **场景 3：层级混淆**
> *示例*：
> ```markdown
> - 若 A 成立则执行 B，若 C 成立则执行 D
> ```
> *问题*：C 是 A 的子条件还是同级条件？AI 无法判断
> *正确*：用缩进区分父子层级

## 常见反模式

### 反模式 1：隐式 else

只有"是"分支，缺少"否"分支。

> **错误**：`若 name 缺失，交互式确认` → 缺了 `是 → 下一步`
> **正确**：`否 -> 通过 AskUserQuestion 提供确认选项（由 AI 根据上下文生成），阻塞等待用户选择` + `是 -> 下一步`

### 反模式 2：内联嵌套

将嵌套条件写在括号或同一行中。

> **错误**：`迁移到 references/ 下（若存在同名文件，交互式确认覆盖、合并或跳过）`
> **正确**：用缩进层级展开：
> ```markdown
> - 迁移到 references/ 下
>     - 是否存在同名文件:
>         - 是 -> 通过 AskUserQuestion 提供选项（覆盖 / 合并 / 跳过），阻塞等待；
>         - 否 -> 下一步；
> ```

### 反模式 3：箭头缺失

有分支内容但未标注 `是 ->` / `否 ->`。

> **错误**：
> ```markdown
> - 检查文件是否存在；
>     - 读取内容；
>     - 提示用户；
> ```
> **问题**：缩进不等于分支，"不存在时"还提示用户吗？
> **正确**：显式标注箭头

## 验证清单

- [ ] 嵌套分支通过缩进正确表达父子层级
- [ ] 每个分支指向具体、可执行的操作指令
- [ ] 不存在括号内嵌套条件（反模式 2）
