# NestJS 文件上传与流处理

## Overview

当 AI 在 NestJS 项目中遇到文件上传相关代码时，自动执行以下工作：审查文件上传器的配置和存储策略，检查文件验证和安全措施，评估流式处理（StreamableFile）的实现，并提供改进建议。

## Definitions

- <a id="目标代码"></a>**目标代码**：当前对话中 NestJS 的文件上传控制器、Multer 配置或流式处理代码。
- <a id="文件拦截器"></a>**文件拦截器**：处理文件上传的 NestJS 内置拦截器，包括 FileInterceptor（单文件）、FilesInterceptor（多文件同名）、FileFieldsInterceptor（多文件异名）和 AnyFilesInterceptor（任意文件）。
- <a id="StreamableFile"></a>**StreamableFile**：NestJS 提供的流式文件响应类，用于高效返回大文件而不占用内存。
- <a id="是否分析完成"></a>**是否分析完成**：标记对目标代码的分析是否已得出完整结果。

## Prerequisites

- NestJS 项目环境；
- 文件上传控制器或配置代码可访问；
- 了解文件上传的基本概念（MIME 类型、大小限制、存储策略）。

## Workflow

0. **前置检查** — 确保目标代码和运行环境可达；
   - 判断目标代码是否存在且可读取：
     - 是 -> 下一步；
     - 否 -> 提示用户提供目标代码或文件路径，阻塞等待用户输入；
   - 初始化全局变量 [是否分析完成](#是否分析完成)：
     - 判断代码是否完整可解析：
       - 满足 -> 初始化变量为 true；
       - 不满足 -> 初始化变量为 false；

1. **分析文件上传代码** — 读取并理解文件处理逻辑；
   - 读取目标代码，识别以下核心要素：
     - 使用的文件拦截器类型（FileInterceptor / FilesInterceptor / FileFieldsInterceptor）；
     - Multer 选项配置（storage、fileFilter、limits）；
     - 文件验证逻辑（文件类型、大小、数量）；
     - 存储策略（本地磁盘 / 云存储）；
     - 流式下载（StreamableFile）的使用；

2. **逐项审查** — 对照审查清单检查文件上传代码质量；
   - 依次判断以下审查项是否通过：
     - 是否配置了文件大小限制（limits.fileSize）避免大文件攻击：
       - 通过 -> 记录通过，继续下一个审查项；
       - 未通过 -> 记录“缺少文件大小限制，建议添加 limits.fileSize”，继续；
     - 是否配置了文件类型过滤器（fileFilter）限制允许的 MIME 类型：
       - 通过 -> 记录通过，继续下一个审查项；
       - 未通过 -> 记录“缺少文件类型过滤器，可能导致不受支持的文件类型被上传”，继续；
     - 存储路径是否使用动态生成的文件名（避免文件名冲突和路径遍历）：
       - 通过 -> 记录通过，继续下一个审查项；
       - 未通过 -> 记录“建议使用 UUID 或时间戳重命名文件替代原始文件名”，继续；
     - 大文件下载是否使用了 StreamableFile（避免将整个文件加载到内存）：
       - 通过 -> 记录通过，继续下一个审查项；
       - 未通过 -> 记录“大文件下载建议使用 StreamableFile 流式返回”，继续；
     - 文件上传端点是否添加了认证/授权守卫（防止未授权上传）：
       - 通过 -> 记录通过，继续下一个审查项；
       - 未通过 -> 记录“文件上传端点缺少访问控制，建议添加守卫保护”，继续；
   - 判断是否有任何问题记录：
     - 是 -> 汇总问题列表，进入下一步；
     - 否 -> 直接进入步骤 4（复核检查）；

3. **提供修改建议** — 对发现的问题给出具体修复方案；
   - 依次对每个问题提供修复建议；
   - 通过 AskUserQuestion 提供选项，阻塞等待用户选择：
     - 采纳全部建议 -> 生成修正后的代码，进入步骤 4；
     - 逐条确认 -> 逐条由用户决定采纳或忽略，全部确认后进入步骤 4；
     - 仅查看不修改 -> 不影响代码，直接进入步骤 4；

4. **复核检查** — 对照 [Review List](#review-list)，确认执行结果；
   - 判断 Review List 是否有内容：
     - 否 -> 直接进入下一步（成果输出）；
     - 是 -> 下一步；
   - 依次判断 [Review List](#review-list) 中每个检查项，是否通过：
     - 基于“复核检查示例”来显示输出内容；
   - 有任一检查失败，终止流程；
   - 全部通过后，进入下一步；

5. **成果输出** — 输出审查摘要，告知完成；
   - 输出结构化摘要表格（审查文件 / 审查项数 / 通过 / 发现问题 / 风险等级 / 已采纳建议）；
   - 告知审查完成；

## Rules

- **内容规范**
  - description 须遵循格式：第一句说明审查内容，第二句说明触发条件，使用第三人称，不超过 1024 字符；
  - 审查项须引用 NestJS 官方文件上传文档；
  - SKILL.md 不超过 300 行；超过时拆分到 references/ 目录；
  - 引用层次不超过一层；
  - 保持术语一致，Definitions 中已定义术语在正文中通过锚点链接引用；
  - 删除任何时效性信息；

- **行为规范**
  - 审查时仅输出问题摘要和定位，不直接修改代码，除非用户通过 AskUserQuestion 明确要求；
  - 所有涉及代码修改的交互环节，必须使用 AskUserQuestion 工具；
  - 存储策略建议（本地 vs 云存储）标注为架构建议而非强制；

- **防御规范**
  - 如目标代码为空或不可读，直接报告并终止；
  - 文件大小限制和类型仅在产品代码中建议，开发环境可放宽；

- **验证规范**
  - Examples 内容必须自洽于 Rules；
  - Examples 必须包含复核检查示例；
  - 对话交互示例仅聚焦步骤 0~3（不含复核检查和成果输出）；

## Examples

### 对话交互示例

**示例：用户请求审查文件上传配置**

```markdown
用户 > 帮我检查文件上传功能的安全性
AI   > 检测到用户需要 NestJS 文件上传审查，触发 nestjs-3-3-file-upload 技能
AI   > 正在分析文件上传代码...

审查结果：
- 🟥 未配置 fileSize 限制，存在大文件攻击风险
- 🟥 未配置 fileFilter，允许所有类型文件上传
- 🟩 使用了 UUID 重命名上传的文件
- 🟥 大文件下载使用了 readFileSync（应使用 StreamableFile）
- 🟩 上传端点添加了 AuthGuard

总评：3 个问题需要关注
用户 > 帮我加固文件上传安全性
AI   > 添加 fileSize: 5MB 限制和图片类型过滤。是否应用修改？
用户 > 全部采纳
```
